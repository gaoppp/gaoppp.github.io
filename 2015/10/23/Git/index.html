Git笔记

发表于 2015-10-23

git初始化

madir test（创建文件夹test） cd test（切换到test目录） touch a.md （新建a.md文件）

git init（初始化仓库）

之后使用 git status 会有提示a.md没有提交到仓库

git提交

git add a.md
使用 git status 会提示a.md文件等待被提交
（可以使用 git rm –cached 清除这个缓存）

git commit -m ‘first commit’
commit——提交 -m——提交信息
使用 git status 将会提示nothing to commit

git log
查看所有产生的commit记录

git分支

git branch 查看分支
git branch a 新建 a分支

git checkout a 切换分支

git checkout -b a 新建分支a，并切换到a

git merge 合并分支

  1. 切换到master分支
  2. git merge a 将a合并到master上

git branch -d a 删除a分支
git branch -D a 强行删除a分支

git设置tag

git tag 用于给代码添加版本tag
git tag v1.0 新建一个tag

git checkout v1.0 切换v1.0 tag的代码状态

git用户名与邮箱

我们每一次commit都会产生一条log 这条log标记了提交人的姓名和邮箱，所有我们在提交代码的第一步是设置自己的用户名与邮箱。以下是执行代码：
git config --global user.name "gaop"
git config --global user.email "gaopdev@gmail.com"

git命令简化与alias（别名）

git commit —— git ci
git checkout —— git co
git branch —— git br
git status —— git st

这是不是特别简洁？这需要以下命令
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status

简化一个特别强大的log命令
git config --global alias.lg "log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative"

之后使用git lg 就行了

其他配置

git config --global core.editor "vim" # 设置Editor使用vim
你们如果喜欢其他编辑器可自行搜索配置，前提是本机有安装。

git config --global color.ui true
开启git着色

git config --global core.quotepath false # 设置显示中文文件名

提交代码到GitHub

  1. 生成SSH key
    ssh-keygen -t rsa 指定rsa算法生成密钥，接着连按3下回车键，就生成了两个文件 id_rsa id_rsa.pub在C:/Documents and Settings/UserName/.ssh
  2. 在GitHub上添加SSH key
    GitHub设置页面找到SSH and GPG keys
    点击 New SSH key将公钥复制过去 点击Add就行了
    添加成功可以输入 ssh -T git@github.com测试

  3. Pull 与 Push
    Push 推送
    git push origin master 将本地代码推送到master分支

    Pull 获取
    git pull origin master 远程的更新到本地

  4. 提交代码
    git clone git@github.com:gaoppp/test.git
    将test项目clone到本地

    git remote add origin git@gitbub.com:gaoppp/test.git
    添加一个远程仓库

    git remote -v
    查看当前项目有哪些远程仓库
